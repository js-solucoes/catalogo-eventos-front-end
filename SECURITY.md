# Política de segurança

## Reportar vulnerabilidades

Não abra **issues públicas** para vulnerabilidades de segurança. Use um canal **privado** acordado pela organização, por exemplo:

- [GitHub Security Advisories](https://docs.github.com/code-security/security-advisories) (dependendo das permissões do repositório), ou
- contacto interno de segurança / responsável pelo produto.

Inclua passos para reproduzir, impacto estimado e, se possível, sugestão de mitigação.

## Versões suportadas

| Ramo   | Suportado |
| ------ | --------- |
| `main` | Sim       |

## Divulgação responsável

Respeite um prazo razoável entre o reporte e a divulgação pública, alinhado com os mantenedores.
